Changelog
=========

All notable changes to vinary-viewer are documented in this file.
The format is based on Keep a Changelog (https://keepachangelog.com/en/1.1.0/),
and this project adheres to Semantic Versioning (https://semver.org/spec/v2.0.0.html).


[0.3.0-dev] - unreleased
------------------------
The common document IR: every supported format now parses into one uniform,
metadata-rich intermediate representation, which lowers to HTML through a single
sanitizer and drives cross-cutting capabilities. See docs/theory/08-common-document-ir.md
and docs/design-decisions/0017-common-document-ir.md.

Added
    - A DOM-free, node-tested common-IR core (vinary.ir.*): a tagged node with
      span/bbox/role/anchor-id metadata; a semiring algebra (Boolean/Tropical/Log/
      Probability/Product/Lexicographic) with full law coverage; a weighted tree
      transducer with exact composition; a weighted pushdown automaton + streaming
      decoder; and an Earley-over-lattice parser with a packed parse forest and
      Viterbi best-parse.
    - Format front-ends parsing into the IR: markdown (rehype HAST), office (docx /
      ODF HTML), source code (web-tree-sitter), PDF (pdf.js text runs → a reflowable
      page/block/line/run facet with bounding boxes), and table / log / archive.
    - Office documents now gain a heading Contents outline and are sanitized by the
      GitHub allowlist (upgrading the previous main-process regex sanitizer), via the
      shared ir/backend/sanitize schema.
    - A font-size heading outline for PDFs that ship no getOutline, plus reading-order
      text extraction for find / copy over the PDF text facet.

    - Source files gain a code Contents outline (from their tree-sitter parse) with
      line-scroll navigation; PDFs without a built-in getOutline gain a font-size
      heading outline (page-scroll navigation).
    - View > Reflow Text: an opt-in view that renders a PDF's extracted text as
      reflowable prose (the page/block/line/run IR -> prose -> HTML), augmenting --
      never replacing -- the faithful canvas render.
    - Full AMS LaTeX family for MathJax: amscd commutative diagrams (\begin{CD}) and
      boldsymbol (\boldsymbol), in addition to the existing amsmath/amssymb + AMS
      fonts.
    - Terminal document previews -- a second renderer over the shared IR / streaming
      spine: `vv --cli <file>` renders once to stdout (pipe-friendly; ANSI text with
      Sixel graphics), and `vv --tui <file>` opens an interactive full-screen pager
      (in-page find, Contents outline, scrolling). The `vv` launcher is now a mode
      dispatcher -- `vv [files…]` opens the desktop GUI (the default; `--gui` is an
      accepted no-op), `vv --cli` / `vv --tui` open the terminal renderers, and
      `vv --help` / `vv --version` print and exit without launching a window.
      install.sh builds and installs all three modes from the one launcher.
    - Emacs Org-mode (.org) support: rendered GitHub-style through the common IR via
      uniorg (uniorg-parse -> uniorg-rehype -> the shared Markdown hast suffix), with
      a heading Contents outline, in-document syntax highlighting for nested
      #+begin_src <lang> blocks, and a tree-sitter-org-highlighted View Source. See
      docs/design-decisions/0020-org-mode-via-uniorg.md and docs/features/26-org-mode.md.

Changed
    - MathJax upgraded to v4 and built from source (@mathjax/src 4.1.3, bundled by
      shadow-cljs) as one engine shared by the renderer and the Node tests, importing
      only the safe TeX packages -- replacing the separately-loaded es5 tex-svg.js
      script. Math typesets in the Latin Modern ("MathJax Modern") font
      (@mathjax/mathjax-modern-font), matching the Latin Modern Roman prose body. The
      dangerous html/require/autoload packages are never imported (absent from the
      bundle), so \href{javascript:...} is unreachable by construction. Assistive
      MathML (a11y) is preserved; math renders synchronously -- MathJax 4's dynamic
      font chunks are preloaded at engine build, so every glyph range renders with no
      async "retry" (variant alphabets \mathbb/\mathfrak/\mathscr/\mathtt/\mathsf,
      extended symbols, arrows, accents, shapes). \llbracket / \rrbracket render as the
      container delimiters ⟦ ⟧ (bound to U+27E6/7 -- MathJax bundles no stmaryrd).
    - The prose / Markdown body font is now Latin Modern Roman (the GUST e-foundry
      successor to Computer Modern -- the LaTeX look), self-hosted as WOFF2 under
      assets/fonts/lm-roman/ (GUST Font License), replacing Noto Sans so body text
      matches the Latin Modern math. Code / logs / tables stay Fira Code.
    - Markdown and office render through the common IR unconditionally; output is
      byte-identical to the retired legacy path (guaranteed by the lossless HAST
      round-trip, ir.parity-test).

Removed
    - The legacy Markdown string renderer, the main-process office regex sanitizer,
      and the :vv/ir migration flag / :ir/set-enabled toggle are retired (kept
      comment-disabled for reference). Office is now sanitized solely by the stronger
      GitHub-allowlist IR schema, with no un-IR path.


[0.2.0] - 2026-07-07
--------------------
A complete rewrite as a self-contained Electron application in ClojureScript /
re-frame, superseding the 0.1.0 vmd-patching approach (preserved at tag v0.1.0).
vinary-viewer no longer depends on vmd: it is a standalone desktop previewer for
Markdown, PDFs, images, source code, Office/OpenDocument files, tables, logs,
archives, and web pages, across multiple tabs, with live refresh.

Added
  Application
    - Standalone Electron app (main = dist/main/main.js), built with shadow-cljs
      (two builds: :main -> Electron main process, :renderer -> Chromium), re-frame +
      DataScript state, React 19 renderer. No longer patches vmd.
    - Multi-tab previews; every file/URI passed on the command line opens in its own
      tab (first focused). Empty-tab watermark (the Vinary Tree shield).
  Document types
    - Markdown (GitHub-Flavored): tables, task lists, strikethrough; stable heading
      anchors; syntax highlighting; GitHub-compatible math typeset to cached MathJax
      SVG; Mermaid diagrams; sanitized raw HTML (rehype-raw + rehype-sanitize, GitHub
      allowlist); copyable LaTeX (Ctrl+C selection rewrite + a "Copy LaTeX" menu item);
      centered, themed-framed standalone figures.
    - PDF: in-renderer pdf.js (canvas + selectable text layer + link layer), windowed
      rendering, zoom / fit / invert, outline-driven scroll-spy TOC, in-page find,
      edge-to-edge layout.
    - Source code: tree-sitter highlighting via a bundled + user grammar registry;
      read-only CodeMirror view.
    - Images: dedicated full-width image view.
    - Content previews: Office / OpenDocument (docx/odt/ods), delimited tables and
      workbooks (paged), large logs (paged), and archives (zip/tar, nested).
    - Web pages / local HTML: a native WebContentsView with native ad-blocking and a
      scoped, GPL-free browser-extension runtime.
  Navigation & reading
    - Git file-tree sidebar with a live filter box; Contents scroll-spy TOC; in-page
      find (CSS Custom Highlight, content-agnostic); navigation history (keyboard,
      mouse thumb buttons, OS app-commands) with scroll restoration; in-pane directory
      browser; breadcrumb + up/down navigation; address-bar path completion; zoom & fit;
      live refresh on save.
    - Themes via CSS-variable palettes (spacemacs-dark default + spacemacs-light),
      live-switchable.
    - Custom keybindings (including Vim-style keymaps); themed dialogs & modals; a
      command palette; a native password-manager bridge (1Password, LastPass).
  Security
    - A renderer Content-Security-Policy (no unsafe-inline script), a navigation
      lock-down (will-navigate / setWindowOpenHandler deny), a pinned MathJax TeX
      package set, and a fail-closed in-content link handler, backing the raw-HTML
      sanitizer.

Changed
    - Architecture: superseded the 0.1.0 vmd-patch model (apply.sh, patch-*.js,
      sidebar.js) with a standalone application; the legacy patch files remain in-tree
      for reference but are not loaded by the running app.
    - Git file-tree now lists tracked AND untracked-but-not-ignored files
      (git ls-files --cached --others --exclude-standard), so newly created files
      appear in the sidebar immediately.

Fixed
    - Markdown math rendered twice (the assistive-MathML copy is now hidden via CSS).
    - Embedded svgbob byte-field SVGs (no root viewBox) rendered as tiny squares; the
      figure sizer now reads the root <svg> dimensions.
    - PDF pages showing white (GPU compositing / willReadFrequently), GPU/Wayland
      scroll bands, raster/remote markdown image scroll-jump, directory-open EISDIR,
      and release-build web-view / PDF / extension crashes.

Notes
    - Platform: Linux + X11 tested (Electron 42). Requires git on PATH for the file tree.


[0.1.0] - 2026-06-23
--------------------
Initial release. vinary-viewer packages a set of enhancements that turn the vmd
Markdown previewer into a full reading environment. It installs by patching vmd's
package in place; the patches survive vmd npm upgrades via a transparent vmd()
shell wrapper that re-applies them on launch.

Added
  Reading environment
    - Git file-tree sidebar of the repository containing the open document, with the
      current file highlighted; hidden entirely when the document is not in a repo.
    - Tabbed sidebar (Files | Contents). The Contents tab is a scroll-spy table of
      contents of the previewed Markdown (click-navigable; disabled for non-Markdown).
    - File-search filter box that narrows the tree to paths matching a query.
    - In-page find bar (Ctrl/Cmd+F): a custom highlighter scoped to the document body
      with match cycling (Enter / Shift+Enter), replacing vmd's broken search webview.
  Layout & figures
    - Full-width layout (removes vmd's 888px centered column; keeps a 45px gutter).
    - Automatic SVG figure font-matching: each embedded SVG is sized so its internal
      text matches the document font (width = docFont x viewBoxWidth / svgFontSize),
      scaling down as well as up; raster images are centered at their natural size.
    - Centered tables.
    - Dedicated full-width image view: opening an image file (or clicking an embedded
      figure) shows just that image.
  Navigation
    - File-history navigation across every open: Alt+Left / Alt+Right (keyboard),
      OS app-command back/forward, and the mouse back/forward thumb buttons.
    - A native X11 addon (mouse-forward-back) grabs mouse buttons 8/9, which Electron 3
      on Linux otherwise swallows before the renderer, and routes them to vmd's history.
  Theming
    - Named color themes via a CSS-variable palette (--vv-*). Bundled: spacemacs-dark
      (default) and spacemacs-light. Select with the VV_THEME environment variable or
      the ~/.config/vinary-viewer/theme file.
  Tooling
    - install.sh / uninstall.sh, the transparent vmd() wrapper, npm scripts (test,
      lint, check, build:addon), a headless test harness, and a theme-completeness lint.

Notes
    - Platform: Linux + X11 (the native addon uses XGrabButton; targets Electron 3 / vmd).
    - vmd is patched in place and is not redistributed; install it separately.
